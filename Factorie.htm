Guideline for Installation and Usage of FACTORIE



Special thanks to Tommy Chheng for sharing his commands and codes for running FACTORIE from command line. Thanks to Abhimanu Kumar for tolerating my Linux-ignorance and helping me generously.

To run FACTORIE from a terminal, first you need to install three packages:
  • Apache maven2: Command- sudo apt-get install maven2
  • mercurial: Command- sudo apt-get install mercurial
  • git: Command- sudo apt-get install git
Now, you can either explicitly download the latest FACTORIE package (0.9.0) from the repository to run it with eclipse or you can use the following commands (I used these commands only) to run FACTROIE from the terminal:

  • Clone the source code: hg clone https://factorie.googlecode.com/hg/ factorie
  • Compile the code into a jar:
    • cd factorie
    • mvn install
    This will create a factorie jar file in target/factorie*.jar and install it into your local maven repository.
  • Download the example directly in your local machine or server from here and save it anywhere. In the example, you’ll notice a pom.xml. Inside the file, you’ll see:
    <dependency>
    <groupId>cc.factorie</groupId>
    <artifactId>factorie</artifactId>
    <version>0.9.1-SNAPSHOT</version>
    </dependency>
    You may have to change the version to match the updated factorie jar version (FACTORIE is going to release version 1.0.0 soon). This sample project has two files:
    • src/main/scala/factorie/LDAExample.scala
    • src/main/scala/factorie/LDAExampleTest.scala
  • Go to the directory where you saved the example project and compile and run it using: mvn test

Tools you need to learn for handling FACTORIE:

  • Scala
  • Maven
You can separately install Scala using the following commands (latest version of FACTORIE uses Scala 2.8.1):
  • wget http://www.scala-lang.org/downloads/distrib/files/scala-2.8.1.final.tgz
  • tar -zxvf scala-2.8.1.final.tgz
  • sudo mv scala-2.8.1.final /usr/share/scala
  • sudo ln -s /usr/share/scala/bin/scala /usr/bin/scala
  • sudo ln -s /usr/share/scala/bin/scalac /usr/bin/scalac
  • sudo rm -rf /usr/share/scala /usr/bin/scala /usr/bin/scalac (for uninstallation)
